請修改 Kiddo Study Planner 的月曆、學習目標與排除日功能。

本專案是 React + Vite 的兒童讀書計畫與自動複習系統，使用 localStorage 儲存資料，不使用後端資料庫，需維持 Vercel 部署相容。

請注意：
不要破壞既有的讀書計畫、複習任務、遺忘曲線、JSON 匯出/匯入功能。

目前系統首頁包含：
1. 今日讀書計畫 / 新學習計畫
2. 要記得複習唷 / 複習任務
3. 你還記得嗎 / 逾期複習任務

目前月曆中已有紅色與綠色標記，但使用者不知道顏色代表什麼，因此需要重新整理月曆標記與說明。

--------------------------------------------------
一、月曆標記說明
--------------------------------------------------

請在月曆上方或右側新增圖例說明：

● 紅色：今日讀書計畫 / 新學習計畫
● 綠色：要記得複習唷 / 複習任務
✕ 排除日

請注意：

1. 紅色代表新學習計畫，不是複習。
2. 綠色代表複習任務，不是新學習。
3. 排除日用 X 表示。
4. 月曆日期格內只需要顯示紅色標記、綠色標記與 X。
5. 月曆日期格內不要直接顯示排除日備註文字。

--------------------------------------------------
二、點選日期後顯示內容
--------------------------------------------------

當使用者點選月曆中的某一天時，請在月曆旁邊或下方顯示該日內容。

請分成兩個區塊，不要混在一起：

1. 當日讀書計畫 / 新學習計畫
   - 顯示該日安排的新學習內容

2. 當日複習任務
   - 顯示該日系統依遺忘曲線產生的複習任務

若該日同時有新學習與複習，兩者要分開顯示。

若該日是排除日，月曆格上只顯示 X，不需要在月曆格內顯示備註。

--------------------------------------------------
三、月曆版面優化：雙月顯示
--------------------------------------------------

目前月曆只顯示當月，右側空白過多，版面浪費。

請改成桌機版雙月顯示。

同時顯示：

1. 本月
2. 下個月

例如：

左側：June 2026
右側：July 2026

目的：

讓使用者可以直接看到未來一個月的新學習計畫與複習安排，方便判斷未來是否排太多讀書計畫。

請注意：

1. 雙月月曆都要顯示紅色、綠色與 X 標記。
2. 本月與下個月都可以點選日期。
3. 點選任一日期，都要顯示該日的新學習計畫與複習任務。

--------------------------------------------------
四、新增「學習目標」功能
--------------------------------------------------

請新增側欄選單：

學習目標

這個功能很簡單，主要讓使用者自己輸入學習目標，並設定預計完成日期。

欄位包含：

1. 目標內容
2. 預計完成日期
3. 是否完成

範例：

- 期中考前完成數學第1～5課
- 暑假完成 Raz Kids Level C
- 7月底前背完10首唐詩
- 完成自然第3單元複習

功能需求：

1. 可以新增學習目標
2. 可以編輯學習目標
3. 可以刪除學習目標
4. 可以標記完成 / 取消完成
5. 學習目標資料需儲存在 localStorage
6. JSON 匯出時需包含尚未刪除的學習目標
7. JSON 匯入時需能還原學習目標

刪除規則：

當使用者按下刪除時，必須跳出確認提醒，避免誤刪。

提醒文字：

確定要刪除此學習目標嗎？
刪除後無法復原。

按鈕：

取消
確認刪除

使用者確認刪除後：

1. 該筆學習目標從 localStorage 永久刪除
2. 不需要封存
3. 不需要回收桶
4. 不需要保留歷程紀錄
5. JSON 匯出時不包含已刪除的學習目標

--------------------------------------------------
五、新增「排除日」功能
--------------------------------------------------

請新增側欄選單：

排除日

用途：

某些日期已經有其他安排，不適合新增讀書進度。

例如：

- 期中考週
- 旅行
- 校外教學
- 家庭活動
- 夏令營
- 比賽週
- 學校活動

排除日不是禁止讀書，而是提醒使用者：這段日期不建議新增讀書進度。

欄位只需要：

1. 開始日期
2. 結束日期
3. 備註

範例：

開始日期：2026/06/20
結束日期：2026/06/27
備註：期中考週，不新增新進度

功能需求：

1. 可以新增排除日
2. 可以編輯排除日
3. 可以刪除排除日
4. 排除日資料需儲存在 localStorage
5. JSON 匯出時需包含尚未刪除的排除日
6. JSON 匯入時需能還原排除日
7. 月曆上排除日區間內的日期需顯示 X

月曆顯示規則：

1. 排除日只在月曆日期格中顯示 X
2. 不要在月曆格內顯示備註文字
3. 如果同一天同時有新學習、複習任務與排除日，可以同時顯示紅點、綠點與 X
4. X 不代表禁止，只代表提醒

刪除規則：

當使用者按下刪除排除日時，必須跳出確認提醒，避免誤刪。

提醒文字：

確定要刪除此排除日嗎？
刪除後無法復原。

按鈕：

取消
確認刪除

使用者確認刪除後：

1. 該筆排除日資料從 localStorage 永久刪除
2. 不需要封存
3. 不需要回收桶
4. 不需要保留歷程紀錄
5. JSON 匯出時不包含已刪除的排除日

--------------------------------------------------
六、新增讀書計畫時的排除日提醒
--------------------------------------------------

當使用者新增讀書計畫時，如果選擇的日期落在排除日區間內，請跳出提醒。

提醒內容：

此日期位於排除日區間：

備註：（顯示該排除日備註）

是否仍要新增讀書計畫？

按鈕：

取消
仍要新增

請注意：

1. 這只是提醒，不是禁止。
2. 如果使用者按「取消」，則不要新增該筆讀書計畫。
3. 如果使用者按「仍要新增」，則照常新增讀書計畫。
4. 不要影響既有的自動複習任務產生規則。

--------------------------------------------------
七、資料保存與刪除原則
--------------------------------------------------

本系統使用 localStorage。

新增的資料包含：

1. 學習目標
2. 排除日

都需納入 localStorage 與 JSON 匯出/匯入。

但只要使用者明確按下刪除，並確認刪除後：

1. 該筆資料需從 localStorage 永久刪除
2. 不需要封存
3. 不需要回收桶
4. 不需要歷程紀錄
5. 不需要已刪除清單
6. JSON 匯出時不包含已刪除資料

--------------------------------------------------
八、保留既有功能
--------------------------------------------------

請務必確認以下功能不受影響：

1. 今日讀書計畫 / 新學習計畫
2. 要記得複習唷 / 複習任務
3. 你還記得嗎 / 逾期複習任務
4. 遺忘曲線規則
5. StudyPlan 資料
6. ReviewTask 資料
7. 是否加入複習計畫的開關
8. localStorage 既有資料
9. JSON 匯出 / 匯入備份
10. 首頁統計與顯示

--------------------------------------------------
九、技術限制
--------------------------------------------------

請維持：

1. React + Vite 架構
2. TypeScript
3. localStorage 儲存
4. 不使用後端資料庫
5. 不使用登入系統
6. 維持 Vercel 部署相容
7. 不使用 Replit 專用 Vite Plugin
8. vite.config.ts 不要讓 Replit 專用設定影響 production build

如果有 Replit 專用 plugin，請確認只在 Replit 開發環境啟用，不要影響 Vercel build。
